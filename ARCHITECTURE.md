# Architecture — ANEX × Rixos промо (v2)

Как устроен лендинг — для того, кто будет его править или переносить в проект Anex.

## Страница сверху вниз

```
<header.hero>             экран-приглашение: лого-локап ANEX×RIXOS, таймер,
                          КАПС-заголовок, подзаголовок, CTA, айфон, плавающие бейджи
<main>
  <section.prizes>        «Что можно выиграть» — 3 карточки призов веером (наклон ±°)
  <section.book>          «Бронируй в RIXOS» — текст + большое фото отеля
  <section.register>      «Регистрируй заявки» — текст + карточка «Заявка отправлена»
  <section.await>         «Ожидай результатов» — айфон + календарь июля в общей сцене
  <section.results>       «Итоги — каждую пятницу» — сетка 8 недель, ближайшая подсвечена
  <section.faq>           аккордеон на нативных <details>
  <section.reg id="reg">  финальная форма — айфон-визуал + поля заявки
<footer>
```

> **Отличие от v1.** v2 показывает шаги участия отдельными двухколоночными секциями (`book`/`register`/`await`: текст слева, медиа справа). v1 те же шаги собирает в один pinned-scrollytelling-блок. Контент и токены общие.

## Дизайн-токены

Все значения вынесены в CSS-переменные в `:root` (`css/styles.css`); имена и комментарии ссылаются на Figma-переменные (node 1931-28124). В стилях секций magic-numbers нет — цвета, радиусы (`--r-8…--r-32`), тени, ширины колонок (`--content`, `--wide`) и семейства шрифтов (`--ff-display/text/num`) идут через токены.

Заголовки используют утилиту `.display` — Roboto Flex с `font-variation-settings` (`wght 800, wdth 143`), имитация платного Druk.

## Механики (`js/app.js`)

Весь скрипт — один IIFE, подключён с `defer` после GSAP-CDN. Уважает `prefers-reduced-motion`.

### Обратный отсчёт — `#clock`
Таймер до ближайшей пятницы (день розыгрыша). Формат `ДД : ЧЧ : ММ : СС`, обновляется ежесекундно.

### Календарь июля — `#calGrid`
Сетка генерится в JS: 1 июля 2026 = среда; пятницы (3, 10, 17, 24, 31) помечаются классом `.fri` и обводятся кружком (SVG-фон в `.fri::before`).

### Reveal, параллакс, конфетти
- `.reveal` — появление секций при скролле (ScrollTrigger добавляет `.in`); большие фото вскрываются clip-path-шторкой.
- Hero-айфон и бейджи — лёгкий параллакс при скролле + плавающая анимация (GSAP `yoyo`).
- Карточки призов — наплывание по очереди (fade, без сдвига наклона веера).
- `[data-confetti]` — запускает canvas-confetti на отправку формы.

### Прогрессивное улучшение
`<body>` стартует с классом `no-js`, который снимается первым же инлайн-скриптом. Если GSAP не загрузился — контент всё равно виден (reveal-элементы не остаются с `opacity:0`).

## Точки интеграции

| Хук | Назначение |
|-----|-----------|
| `[data-scroll="#reg"]` | плавный скролл к форме по клику |
| `[data-confetti]` | триггер конфетти |
| `#clock` | контейнер таймера (заполняется скриптом) |
| `#calGrid` | контейнер календаря (генерится скриптом) |
| `#regForm` | форма заявки — submit перехвачен (`preventDefault` + конфетти); **на проде подключить реальный обработчик** |

## Чего здесь нет (сознательно)

- Сборки/бандлера — статика разворачивается копированием папки.
- Фреймворка — vanilla JS, чтобы разработчики Anex встроили в любой стек.
- Реального бэкенда формы — это макет-прототип под передачу.
