# ANEX × Rixos — промо «Получи айфон за бронь тура» (v2)

B2B-лендинг розыгрыша для турагентов. Агент бронирует тур в **Rixos Hotels (Египет)** → регистрирует номер заявки → каждую пятницу разыгрываются **iPhone 17 Pro**, сертификаты $2500 и выездные воркшопы.

**Версия 2** — карточки призов веером, КАПС-заголовок. Свёрстано по Figma `node 1931-28124`.
Отличия от v1: веер призовых карточек, заглавный заголовок hero, отдельные двухколоночные секции «Бронируй / Регистрируй / Ожидай» (текст слева, медиа справа).

## Стек

Чистый **HTML / CSS / JS, без сборки**. Внешние зависимости — только CDN:

- [GSAP](https://gsap.com/) + ScrollTrigger — анимации, параллакс, reveal
- [Lenis](https://lenis.darkroom.engineering/) — плавный скролл
- [canvas-confetti](https://github.com/catdad/canvas-confetti) — конфетти на отправку формы

## Запуск локально

Сборки нет. `css/` и `js/` подключаются по относительным путям, поэтому открывать через статический сервер (а не `file://`):

```bash
python3 -m http.server 8000
# → http://localhost:8000
```

## Интеграция в проект Anex

- **Разметка** — обычные секции внутри `<main>`, переносятся как есть.
- **Дизайн-токены** — все цвета, радиусы, тени, ширины и шрифты вынесены в CSS-переменные в `:root` (`css/styles.css`) и подписаны соответствием Figma-переменным. Magic-numbers в стилях секций нет.
- **JS самодостаточен** — навешивается на `id`/`data`-атрибуты: `#clock`, `#calGrid`, `[data-scroll]`, `[data-confetti]`, `#regForm`. Точки интеграции описаны в [ARCHITECTURE.md](./ARCHITECTURE.md).
- **Шрифт заголовков** — Roboto Flex (вариативный, `wdth 143%`) как бесплатный стенд-ин платного **Druk**; на проде заменить на Druk.

## Перед запуском в прод

- Фото отелей Rixos — запросить **медиа-кит** у партнёра (текущие — плейсхолдеры, права).
- Даты пятниц и счётчики призов — демо-значения, выверить с организатором.
- Подключить реальный обработчик отправки формы `#regForm` (сейчас — только конфетти).
